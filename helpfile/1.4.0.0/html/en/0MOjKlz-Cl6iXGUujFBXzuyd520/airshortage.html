AirShortage (FB)¶

FUNCTION_BLOCK AirShortage

Short Description¶

Detection of an air shortage situation and generation of a fault message

Typical application: Air deficiency monitoring on a fan with a differential pressure switch

Functional description¶

General¶

This function block is used for detection of an air shortage situation in a fan by evaluating the signals at the inputs
xPFb input - differential pressure switch and xFanFb operating message - fan.

Ausgang - Störmeldung xAlarm¶

The xAlarm output - fault message is activated (xAlarm = TRUE) if conditions 1 and 2 below are fulfilled at the same time.

Condition 1: The xPFb input for the differential pressure switch is permanently in the state FALSE (= air shortage situation) at least during the udiControlTime waiting time.
At the same time, the monitoring is generally enabled ( **xControl**= TRUE).

Condition 2: The operating message for the fan xFanFb is permanently in the state TRUE (= fan in operation) at least during udiControlTime waiting time.
At the same time, the monitoring is generally enabled ( **xControl**= TRUE).


The *xAlarm*output - fault message is deactivated in the following cases (*xAlarm*= FALSE):

Case 1: Conditions 1 and / or 2 are not fulfilled and the reset of the *xQuit*fault message is active (= TRUE).

Case 1: Conditions 1 and / or 2 are not fulfilled and the enable input xControl for the monitoring is inactive ( = FALSE).

tTimeToAlarm time period until the fault message is triggered¶

Depending on the operating situation, different values ​​are displayed at the output *tTimeToAlarm*time until the fault message is triggered.

Situation 1: Output of the remaining time until the fault message is triggered (conditions: xPFb = FALSE, **xControl**= TRUE, xFanFb = TRUE)

Situation 2: If situation 1 is not present, the value 0.0 is permanently output.
