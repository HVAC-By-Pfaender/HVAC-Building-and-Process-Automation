Short Description¶

Typical application: Control valve in a heating circuit

Functional description¶

rAA control signal before manual override¶

The rAA control signal before manual override is influenced by the blocking protection module. The blocking protection module is parameterised by the
udiAbsTime, tAbsStartTime und eAbsDay set-points/ parameters, the *rU*position feedback from the control valve being used as input signal.
The minimum change in valve position to lock the blockin protection function is fixed at 10.0%.
The rAA control signal before manual override is limited by ​​**rAbsMinLimit** and rAbsmaxLimit, the two absolute limit values.
The rU position feedback from the control valve can be replaced by the rY control signal from regulation if no position feedback is available.

rHA control signal after manual override¶

The rHA control signal after manual override is influenced by the output of the anti-lock module, the rY control signal from regulation and the manual override.

position monitoring¶

The position monitoring can be enabled (**xControl**= TRUE) or disabled (**xControl**= FALSE).
When position monitoring is enabled, the value of the difference between the rHA control signal after manual override and the rU position feedback from the control valve is permanently determined
and compared to the **rControlDiff**limit value. If this difference permanently exceeds the limit value during a **udiControlTime**message delay, an alarm signal is generated if
at the same time the feedback - automatic operation *byAU*signals the automatic operation (*byAU*= 0). This alarm signal is displayed with the highest priority at the sStatus output in plain text
and at the same time the xAlarm collective fault message (*xAlarm*= TRUE) is transmitted. The alarm signal is deactivated again (*xAlarm*= FALSE) as soon as the difference
falls again below the limit value

rU position feedback from the control valve¶

The rU position feedback from the control valve can be replaced by the rY control signal from regulation if no position feedback is available.

blocking protection module¶

The blocking protection module is implemented by the numerical blocking protection module with adjustable time duration (FB: AbsA).
The udiAbsTime time period, the todAbsStartTime point in time and the eAbsDay weekday are adjustable.
The minimum run time for locking the blocking protection operation is 30s.
The dtAbsLastOn time stamp (date / time) of the last anti-lock operation is available.
