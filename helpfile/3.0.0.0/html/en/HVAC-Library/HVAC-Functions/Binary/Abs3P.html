Functional Description¶

General¶

This function block is used to monitor message inputs xFbOpen and xFbClose to activity. The test period covers 168 hours (= 1 week).
The message inputs xFbOpen and xFbClose are active, if they are continuously active during the minimum operating time tMinRunTime and also during the test period (only xFbOpen or only XFbClose or both).
The anti-lock process is thus prevented, the test time restarting.

If no activity was detected during the test period, the anti-lock process is triggered on the weekday eDay at todStartTime time.
At the same time, the test period restarts.

The active anti-lock process is divided into two consecutive periods of time (Section 1 and Section 2)
In period 1, the combination Enable - Output xOutOpen = TRUE, Enable - Output xOutClose = FALSE is shown during the time duration tTime.
In period 2, the combination Enable - Output xOutOpen = FALSE, Enable - Output xOutClose = TRUE is shown during the time duration tTime.

The initial triggering of the anti-lock process may be delayed depending on the relevant settings ( weekday eDay and todStartTime ).

Example

Testing period completed on Tuesday morning, start of the anti-lock process not until next Monday.

At the dtAbsLastOn output, the start date and the start time of the last anti-lock process are available.

Prerequisite for the use of the Abs3P functional component

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Display of the anti-lock process xAbsOn¶

This output is used to display an active anti-lock process (xAbsOn = TRUE).

Setpoint / Parameter eDay¶

The anti-lock function can be disabled by set the setting to eDay = eDoW.Inactive.

Setpoint / Parameter**tTime**¶

The anti-lock function can be disabled by set the setting to tTime = 0s.
