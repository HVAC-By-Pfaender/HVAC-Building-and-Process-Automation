Functional Description¶

General¶

The function block is basically used to define the operating modes night mode ( xQ = FALSE ) and day mode ( xQ = TRUE ) depending on the states at the inputs time program ( xClockchannel ) and operating button ( xButton and/or xButtonVisu ).
The operating behavior is influenced by a total of two adjustable options.

The basic functions are available in the basic configuration section (see below for setting the options).

Enable output xQ¶

The enable output xQ always indicates the current state of the switching element depending on the inputs and the two options.

Display useful time extension xQET¶

The display output xQET always indicates the active state of the operating time extension.

Remaining running time - operating time extension tET¶

The display output tET always shows the current remaining time of an active operating time extension.
Immediately after the first download and before the first start of the timer, the value T#0s is always displayed.

Option tEnClockButton¶

This option locks or unlocks the button during the active time program.
If the setpoint / parameter tEnClockButton is TRUE, then with the Buttons (xButton and/or xButtonVisu) the output xQ can be switched off and on when the input xClockchannel is activated.

Option tEnToggle¶

If the setpoint / parameter tEnToggle is TRUE, then the operating time extension can be switched on and off in night operation with the Buttons (xButton and/or xButtonVisu). Otherwise, it is only possible to start the operating time extension.

Scaling / display of time duration udiTime¶

The display output tTime permanently shows the current value of the setpoint / parameter udiTime in scaled form ( eTimeScale )
and absolutely limited by tMinTime or tMaxTime.

Display of the lower time limit udiMinVisu¶

The display output udiMinVisu permanently shows the current value of the setpoint / parameter tMinTime in scaled form ( eTimeScale )
and absolutely limited by tMinTime or tMaxTime.

Display of the lower time limit udiMaxVisu¶

The display output udiMaxVisu permanently shows the current value of the setpoint / parameter tMaxTime in scaled form ( eTimeScale )
and absolutely limited by tMinTime or tMaxTime.
