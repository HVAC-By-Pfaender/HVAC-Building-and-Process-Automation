Burner3P (FB)¶

FUNCTION_BLOCK Burner3P

Short Description¶

Control of a heat producer ( temperature control via binary control commands More / Less )

Typical application: heat production

Portrayal¶

Functional Description¶

General¶

This function block is used to control a heat producer depending on the requirements of the heat Producer xEn, the flow setpoint temperature rSetPtTemp and the flow temperature rFdT.
The heat producer is enabled / disabled via output xHB ( FALSE = disable, TRUE = enable ).
The flow temperature control ( analog also boiler temperature control ) is carried out by means of two power controllers ( P controller, slow and fast ) through the outputs xMoreHB and xLessHB.
The control is enabled with a time delay ( setpoint / parameter udiWaitControl ) after the operating feedback ( xFb = TRUE ) of the heat producer.
Essentially, the following function blocks are used:

ControlP

Out3P

OperatingHours

ManBin

Enabling / disabling of the heat producer¶

The heat producer is enabled ( xAB = TRUE, xHB = TRUE ( eManModeB = eManBin.Auto )) if the flow temperature rFdT falls below the sum of the flow setpoint temperature rSetPtTemp and setpoint / parameter rSwitchOnPoint, the general enable xEn is active ( = TRUE ) at the same time and the minimum downtime is not active.
The heat producer is disabled ( xAB = FALSE, xHB = FALSE ( eManModeB = eManBin.Auto )) if the flow temperature rFdT exceeds the sum of flow setpoint temperature rSetPtTemp and setpoint / parameter rSwitchOffPoint and the general enable xEn is active ( = TRUE ) at the same time.
Between two consecutive operating cycles, a minimum idle time is always inserted ( duration = udiTimeMinOff, display - remaining time: tOnDelay ).
The minimum idle time begins with the falling edge ( TRUE > FALSE ) at input xFb.

Temperature control of the heat producer¶

The temperature control is enabled after a time delay ( duration = udiWaitControl, display - remaining time: tOnControlDelay ).
after the active operating message ( xFb = TRUE ) of the heat producer.
The temperature control is disabled immediately after the inactive operating message ( xFb = FALSE ) of the heat producer ( switch-off value: rYDisabled ).
The two internal power controllers ( P controllers, slow and fast, output range -100.0 - 100.0 % each ) are enabled / disabled simultaneously. ( Set point = flow set point temperature rSetPtTemp, actual value = flow temperature rFdT ).
The control signals of both power controllers are supplied to the output controller in the range -100.0 - 100.0 % in minimum value selection ( transmission of the signal with the lower value ).
The output controller then generates output signals for a 3-point drive ( pulse width modulation, integration interval = tIntegration ) with the outputs xMoreHB or xLessHB.
The constant impulse duration for the two outputs is freely adjustable ( tImpulsOpen or tImpuls Close ), the variable pulse pauses are defined by the output controller.
The two power controllers can be configured individually and dependent on each other ( P controller fast: gain factor rKpBig, P controller slow: gain factor rKpTiny ).

Manual override of the digital outputs¶

All digital control outputs after manual override ( xHB, xMoreHB*, xLessHB ) can be operated either in automatic mode ( eMANBIN.Auto ) or in one of the two manual modes ( eMANBIN.ManOn, eMANBIN.ManOff ).

xAlarm collective alarm¶

The collective Alarm message xAlarm is active ( = TRUE ) if the fault message of the heat generator xAlarmIn is active ( = TRUE ).
In all other cases, the collective fault xAlarm is not active (= FALSE).

Collective alarm message automatic mode xAuto¶

The collective alarm message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

- Eingang eAOO = HVACTYPES.eManBin.Auto
- Eingang eAOOMore = HVACTYPES.eManBin.Auto
- Eingang eAOOLess = HVACTYPES.eManBin.Auto
- Sollwert / Parameter eManModeB = eMANBIN.Auto
- Sollwert / Parameter eManModeBMore = eMANBIN.Auto
- Sollwert / Parameter eManModeBLess = eMANBIN.Auto

Operating hours counter - heat producer¶

The operating hours counter ( udiOperatingHours, udiCountFb ) for the heat producer is active when the operating message is active ( xFb = TRUE ).

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Prerequisite for using the Burner3P function block

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Current calculated power of the heat generator rPos¶

The current calculated power of the heat producer is displayed at output rPos in the range 0.0 - 100.0 %.
This requires the correct specification of the running time of the drive between minimum and maximum power ( tDuration ).
There is no repositioning at a reference position.

Recirculation of the heat producer output rU¶

The current actual power of the heat producer can be connected to the rU input.
This input is currently not used for control purposes.
