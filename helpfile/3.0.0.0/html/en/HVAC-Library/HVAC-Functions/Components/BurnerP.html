BurnerP (FB)¶

FUNCTION_BLOCK BurnerP

Short Description¶

Control of a heat producer ( temperature control via analoge power signal 0.0 - 100.0 % )

Typical application: heat production

Portrayal¶

Functional Description¶

General¶

This function block is used to control a heat producer depending on the requirements of the heat Producer xEn, the flow setpoint temperature rSetPtTemp and the flow temperature rFdT.
The heat producer is enabled / disabled via output xHB ( FALSE = disable, TRUE = enable ).
The flow temperature control (analog also boiler temperature control) is carried out by means of a power controller (PI controller) through the analog output rHa in the range from rYMin to rYMax.
The control is enabled with a time delay ( setpoint / parameter udiWaitControl ) after the operating feedback ( xFb = TRUE ) of the heat producer.
Essentially, the following function blocks are used:

ControlPI

OperatingHours

ManBin

ManNum

Enabling / disabling of the heat producer¶

The heat producer is enabled ( xAB = TRUE, xHB = TRUE ( eManModeB = eManBin.Auto )) if the flow temperature rFdT falls below the sum of the flow setpoint temperature rSetPtTemp and setpoint / parameter rSwitchOnPoint, the general enable xEn is active ( = TRUE ) at the same time and the minimum downtime is not active.
The heat producer is disabled ( xAB = FALSE, xHB = FALSE ( eManModeB = eManBin.Auto )) if the flow temperature rFdT exceeds the sum of flow setpoint temperature rSetPtTemp and setpoint / parameter rSwitchOffPoint and the general enable xEn is active ( = TRUE ) at the same time.
Between two consecutive operating cycles, a minimum idle time is always inserted ( duration = udiTimeMinOff, display - remaining time: tOnDelay ).
The minimum idle time begins with the falling edge ( TRUE > FALSE ) at input xFb.

Temperature control of the heat producer¶

The temperature control is enabled with a time delay ( duration = udiWaitControl, display - remaining time: tOnControlDelay ) after the active operating message ( xFb = TRUE ) of the heat generator.
The temperature control is disabled immediately after the inactive operating message ( xFb = FALSE ) of the heat producer ( power signal - heat producer before manual override rAA in disabled state = rYDisabled ).
The internal power controller ( PI controller, output range: rYMin to rYMax ) is enabled / disabled with the temperature control. ( Set point = flow set point temperature rSetPtTemp, actual value = flow temperature rFdT ).
The parameters of the internal power controller can be configured individually:

Power controller - Upper absolute limit value of the output signal in % : rYMax
Power controller - Lower absolute limit value of the output signal in % : rYMin
Power controller - integral time in s : rTn
Power controller - amplification factor : rKp
Power controller - offset temperature in °C : rOffset
Power controller - Initialization value for the output signal in % : rInitValue
Power controller - switch-off value in % : rYDisabled

The output signal of the power controller is supplied directly to the output rAA.

xAlarm collective alarm¶

The collective alarm xAlarm is active ( = TRUE ) if the fault message of the heat generator xAlarmIn is active ( = TRUE ).
In all other cases, the collective alarm fault xAlarm is not active (= FALSE).

Collective message automatic mode xAuto¶

The collective message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

- Eingang eAOO = HVACTYPES.eManBin.Auto
- Eingang eAO = HVACTYPES.eManNum.Auto
- Sollwert / Parameter eManModeB = eMANBIN.Auto
- Sollwert / Parameter eManModeN = eMANNUM.Auto

Operating hours counter - heat producer¶

The operating hours counter ( udiOperatingHours, udiCountFb ) for the heat producer is active when the operating message is active ( xFb = TRUE ).

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Prerequisite for using the BurnerP function block

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Recirculation of the heat producer output rU¶

The current actual power of the heat producer can be connected to the rU input.
This input is currently not used for control purposes.

Feedback of the manual value for the power signal from the manual override - Hardware rPoti¶

The manual value feedback can be connected to the rPoti input.
This input is currently not used for control purposes.
