DuoPump (FB)¶

FUNCTION_BLOCK DuoPump

Short Description¶

Redundant control of two similar circulation pumps with anti-lock function
Additional functions: Requirements for power generation, manual override, pump overrun, operating hours counter, operation monitoring
Typical application: Redundant control of a double pump in one heating circuit

Functional Description¶

General¶

This function block controls two similar circulation pumps in redundant operation.
The number of required circulation pumps is selected via the inputs xEnOne and xEnTwo.
The circulation pumps 1 and 2 are enabled via the outputs xHB1 and xHB2 or xAB1 and xAB2.

Essentially, the following function blocks are used:

Redundance2

AbsB

OperatingHours

ManBin

Prerequisite for using the DuoPump function block

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Redundant control¶

It is used for redundant control of the two circulation pumps depending on the influencing variables requirements ( xEnOne and xEnTwo ), operating hours ( uxiOperatingHours1 and uxiOperatingHours2 ), operating states ( xFb1 and xFb2 ), alarm states ( xAlarm1 and xAlarm2 ) and the operating mode for the sequence definition ( eModeRedundance ).
The release of the master pump and, if required, of the slave pump is determined internally and transferred to the internal outputs xCmd1 or xCmd2. It takes account to the the current sequence.

The lead pump is required if it is requested by the xEnOne input and / or while the pump run-on time is active.
The lead and follower pumps are required when requested by the xEnTwo input.
Requirements at input xEnTwo have priority over requirements at input xEnOne.

The master or slave pump is selected with high priority via the states of the alarm messages of the circulation pumps xAlarm1 or xAlarm2 ( Alarm = TRUE ).

The master or slave pump is selected with low priority via the eModeRedundance parameter.
If eModeRedundance = Auto, the selection is made via the current number of operating hours of the circulation pumps.
The time difference for switching over the operating hours is 100 hours. The switching is done according to different criteria, which are described in the Function block Redundance2.
The options xConOperatingHours and xEnFb are enabled (= TRUE).
In the case of eModeRedundance = ForceComponent1 or eModeRedundance = ForceComponent2, the guide pump is the same as circulation pump 1 or circulation pump 2.

operating messages xFb1 and xFb2¶

Circulation pump 1 in operation: xFb1 = TRUE
Circulation pump 1 not in operation: xFb1 = FALSE

Circulation pump 2 in operation: xFb2 = TRUE
Circulation pump 2 not in operation: xFb2 = FALSE

Alarm messages xAlarmIn1 and xAlarmIn2¶

Circulation pump 1 faulty: xAlarmIn1 = TRUE
Circulation pump 1 not faulty: xAlarmIn1 = FALSE

Circulation pump 2 faulty: xAlarmIn2 = TRUE
Circulation pump 2 not faulty: xAlarmIn2 = TRUE

Resetting the alarm signal of the operational monitoring systems xQuit¶

The resetting of the alarm message of the operation monitors xAlarmFb1 and xAlarmFb2 takes place as long as the resetting of the alarm message of the operation monitors xQuit is active ( = TRUE ).

Requirement - Power generation xEnergyRequest¶

The requirement - energy production xEnergyRequest is influenced by the anti-lock modules. the follow-up modules and the operating feedback signals of circulation pumps 1 and 2 xFb1 and xFb2.

The requirement - energy production xEnergyRequest is active if

The operating message xFb1 from circulation pump 1 is present and the anti-lock module 1 is not active at the same time.
and / or
the operating message xFb2 from circulation pump 2 is present and the anti-lock module 2 is not active at the same time
and
the overrun module is not active

Follow-up period¶

The overrun module is implemented by a switch-off delay with adjustable duration udiOffDelay. The overrun module is activated if there is no more request via the inputs xEnOne or xEnTwo.
The pump overrun only takes place at the guide pump.

anti-lock modules¶

The anti-lock modules are implemented by the binary anti-lock module with adjustable duration.
The duration tAbsTime, the time tAbsStartTime and the weekday eAbsDay can be set for the first pump.
The time for the second pump is the time of the first pump offset by twice the duration of the first pump tAbsTime.
The time stamp (date/time) of the last anti-lock operation is available separately for each circulation pump at the outputs dtAbsLastOn1 or dtAbsLastOn2.

Collective alarm - Circulation pump 1 xAlarm1¶

The collective alarm - circulating pump 1 xAlarm1 is active ( = TRUE ), if the alarm message of circulating pump 1 xAlarmIn1 is active ( = TRUE ) and / or a alarm signal - operation monitoring - circulating pump 1 xAlarmFb1 ( = TRUE ) is present.
In all other situations, the collective message - circulation pump 1 xAlarm1 is not active (= FALSE).

Collective message - Circulation pump 2 xAlarm2¶

The collective alarm - circulation pump 2 xAlarm2 is active ( = TRUE ), if the alarm message of the circulation pump 2 xAlarmIn2 is active ( = TRUE ) and / or a alarm message - operation monitoring - circulation pump 2 xAlarmFb2 ( = TRUE ) is present.
In all other situations, the collective alarm - circulation pump 2 xAlarm2 is not active (= FALSE).

xAlarm collective alarm¶

The collective alarm xAlarm is active ( = TRUE ) if the collective alarm- circulation pump 1 xAlarm1 is active or the collective alarm - circulation pump 2 xAlarm2 is active.
In all other cases, the collective alarm xAlarm is not active (= FALSE).

Operating hours counter - circulation pump 1¶

The operating hours counter for circulation pump 1 is active if the operating message xFb1 = TRUE and the collective alarm xAlarm1 = FALSE are both active.

Operating hours counter - circulation pump 2¶

The operating hours counter for circulating pump 2 is active if the operating message xFb2 = TRUE and the collective alarm xAlarm2 = FALSE at the same time.

Write access to the counter readings

Via the path ‘Instancename._OperatingHours[1].udiOperatingHours’ or ‘Instancename._OperatingHours[1].udiCountFb’ a write access to both counter readings is possible.
Via the path ‘Instancename._OperatingHours[2].udiOperatingHours’ or ‘Instancename._OperatingHours[2].udiCountFb’ a write access to both counter readings is possible.

Operation monitoring - Circulation pump 1¶

The operation monitoring monitors the correct function of the circulation pump 1.
The monitoring can generally be enabled (xFbControl = TRUE) or disabled (xFbControl = FALSE).
The operating mode of the manual override - hardware circulation pump 1 must be in automatic mode (eAOOO1 = HVACTYPES. eManBin. Auto).

The alarm message - operation monitoring - circulation pump 1 xAlarmFb1 is activated ( = TRUE ) if during the period of operation monitoring udiFbControlTime the states of the input xFb1 and the output xHb1 are unequal.

The alarm message - operation monitoring - circulation pump 1 xAlarmFb1 is deactivated ( = FALSE ) if at least one of the following conditions is active:

eAOO1 = HVACTYPES.eManBin.Auto (Switch to automatic mode of manual override - Hardware).
Reset of the alarm message of the operation monitoring xQuit = TRUE
Release of the operation monitoring xFbControl = FALSE

Operation monitoring - Circulation pump 2¶

The operation monitoring monitors the correct function of the circulation pump 2
The monitoring can generally be enabled (xFbControl = TRUE) or disabled (xFbControl = FALSE).
The operating mode of the manual override - hardware circulation pump 2 must be in automatic mode (eAOO2 = HVACTYPES. eManBin. Auto).

The alarm message - operation monitoring - circulation pump 2 xAlarmFb2 is activated ( = TRUE ) if during the period of operation monitoring udiFbControlTime the states of the input xFb2 and the output xHb2 are unequal.

The alarm message - operation monitoring - circulation pump 1 xAlarmFb1 is deactivated ( = FALSE ) if at least one of the following conditions is active:

eAOO2 = HVACTYPES.eManBin.Auto (Switch to automatic mode of manual override - Hardware).
Reset of the alarm message of the operation monitoring xQuit = TRUE
Release of the operation monitoring xFbControl = FALSE

Collective message automatic mode xAuto¶

The collective message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

- Eingang eAOO1 = HVACTYPES.eManBin.Auto
- Eingang eAOO2 = HVACTYPES.eManBin.Auto
- Sollwert / Parameter eManModeB1 = eMANBIN.Auto
- Sollwert / Parameter eManModeB2 = eMANBIN.Auto
