Fan1Speed (FB)¶

FUNCTION_BLOCK Fan1Speed

Short Description¶

Control of a single-stage fan
Additional functions: low air pressure monitoring, operation monitoring, repair switch, manual override, status display
Typical application: control of a room - exhaust fan

Functional Description¶

General¶

This function block controls a single-stage fan.
Essentially, the following function blocks are used:

AirShortage

ManBin

OperatingHours

Enable - Fan before manual override xAB¶

The release - fan before manual override xAB is influenced by the request of the fan xEn, the collective alarm message xAlarm, the message of the repair switch xMSwitch, the parameter xAlarmControl and the air shortage monitoring.

The enable fan before manual override xAB is active ( = TRUE ) if the following states are present simultaneously:

Case 1:

xEn = TRUE ( request active )
xMSwitch = TRUE ( Motor protection switch not tripped )
xAlarmControl = FALSE ( collective Alarm message not considered )

Case 2:

xEn = TRUE ( request active )
xAlarm = FALSE ( collective alarm message not active )
xMSwitch = TRUE ( Motor protection switch not tripped )
xAlarmControl = TRUE ( collective alarm message not considered )

In all other situations the enable - fan before manual override xAB is not active ( = FALSE ).

Enable - Fan after manual override xHB¶

The release - fan before manual override xHB corresponds to the release - fan before manual override xAB, additionally extended by a manual override module and a forced override.

Repair switch message xMSwitch¶

xMSwitch = TRUE: Repair switch was not triggered ( normal state )
xMSwitch = FALSE: Repair switch was triggered ( error condition )

Forced override xForce¶

The forced override acts directly on the output xHB. The parameter xForceOption defines the state of the output xHB when TRUE at the input xForce.

Operating hours and switch-on procedures¶

The number of operating hours and the number of switch-on processes are recorded ( = counting process active ) if the fan operating message xFb is active and the collective alarm message xAlarm is not active ( = FALSE ) at the same time.
The results are available at the counter outputs udiOperatingHours and udiCountFb.

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Date / time of the last operation of the fan dtLastFb¶

The output dtLastFb provides the current date/time during the counting process.
At the output dtLastFb, the date/time at the end of the last counting process is available outside the counting process.
The output dtLastFb outputs the initialization value DT#1970-1-1-1-0.0.0 before the first counting process.

Operation monitoring¶

Operation monitoring can be enabled ( = TRUE ) or disabled ( = FALSE ) by the setpoint / parameter xFbControl.

If monitoring is disabled, the alarm message of the operational monitoring xAlarmFb is always disabled ( = FALSE ).
When monitoring is enabled, the alarm message of the operational monitoring xAlarmFb is activated ( = TRUE ) if the following states are simultaneously present during a minimum duration udiFbControlTime:

xHB and xFb = unequal ( release state and operating state do not match )
eAOO = HVACTYPES.eManBin.Auto ( Manual override - Hardware in automatic )
xFbControl = TRUE ( Operation monitoring is enabled )

The alarm message of the operation monitoring xAlarmFb is reset ( = FALSE ) if at least one of the following conditions is fulfilled:

eAOO = HVACTYPES.eManBin.Auto ( switch back to automatic mode of Manual override - hardware )
xQuit = TRUE ( reset of error Messages is active )
xFbControl = FALSE ( Operation Monitoring disabled )

low air pressure monitoring¶

It is used to detect an air shortage situation by evaluating the signals at the inputs, the signal of the air shortage situation xPFb and the operating signal of the fan xFb.
The low air pressure monitoring can be enabled ( = TRUE ) or disabled ( = FALSE ) by the setpoint / parameter xPFbControl. In the locked state, the output error message xAlarmPFb is not activated ( = FALSE ).

The output alarm message xAlarmPFb is activated ( xAlarmPFb = TRUE ) if the conditions 1 and 2 below are fulfilled simultaneously.

Condition 1: The input for the differential pressure switch xPFb is permanently in the state FALSE ( = air shortage situation ) at least during the waiting time udiPFbControlTime.

Condition 2: The operating message for the fan xFb is permanently in the TRUE state (= fan in operation) at least during the udiPFControlTime waiting time.


The output alarm message xAlarmPFb is deactivated in the following cases ( xAlarmPFb = FALSE ):

Case 1: Conditions 1 and / or 2 are not fulfilled and the reset of the *xQuit*fault message is active (= TRUE).

Collective alarm signal xAlarm¶

The collective alarm signal is activated ( = TRUE ) if at least one of the conditions described below is fulfilled.

xAlarmIn = TRUE ( Fan fault message is active )
xAlarmFB = TRUE ( error from Operation monitoring )
xAlarmPFb = TRUE ( error from low air pressure Monitoring )
xMSwitch = FALSE ( repair switch was triggered )

In all other cases, the collective alarm is not active (= FALSE).

Collective message automatic mode xAuto¶

The collective message Automatic mode xAuto is activated ( = TRUE ) if the input eAOO contains the value HVACTYPES.eManBin.Auto and the setpoint / parameter eManModeB contains the value eMANBIN.Auto.
In all other cases, the collective message Automatic mode xAuto is not activated ( = FALSE ).
