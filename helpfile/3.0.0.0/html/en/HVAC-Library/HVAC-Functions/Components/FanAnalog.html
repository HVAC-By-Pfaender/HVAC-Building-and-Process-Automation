FanAnalog (FB)¶

FUNCTION_BLOCK FanAnalog

Short Description¶

Control of a variable speed fan
Additional functions: Manual override, repair switch, operating hours counter, air shortage
Typical application: Control of a fan with frequency converter

Functional Description¶

General¶

This function block controls a variable-speed fan via a continuous control signal rHA, which in automatic mode ( eManModeN = eManNum.Auto ) is taken over from input rY.
The general release of the fan takes place via the request xEn and the release signal xHB.
Essentially, the following function blocks are used:

ManBin

ManNum

AirShortage

CTime

OperatingHours

Enable - Fan before manual override xAB¶

The release - fan before manual override xAB is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

Case 1:

xEn = TRUE
xMSwitch = TRUE
xAlarmControl = FALSE

Case 2:

xEn = TRUE
xMSwitch = TRUE
xAlarmControl = TRUE
xAlarm = FALSE

In all other situations the Release of the fan, before manual override xAB, is not activated ( = FALSE ).

Enable - Fan after manual override xHB¶

The release - fan after manual override xHB corresponds to the release - fan before manual override xAB, additionally extended by a manual override module.
The release can generally only take place if the forced locking of the fan is not active after manual override ( xForce = FALSE ).
and the collective alarm signal is not active ( xAlarm = FALSE ).

Repair switch message xMSwitch¶

xMSwitch = TRUE: Repair switch was not triggered ( normal state )
xMSwitch = FALSE: Repair switch was triggered ( error condition)

Forced shut-off of the fan after manual override xForce¶

xForce = FALSE: no foced locking ( normal state )
xForce = TRUE: forced locking ( state of emergency )

Operating hours and switch-on procedures¶

The number of operating hours and the number of switch-on processes are recorded ( = counting process active ) if the fan operating message xFb is active and the collective alarm message xAlarm is not active ( = FALSE ) at the same time.
The results are available at the counter outputs udiOperatingHours and udiCountFb.

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Prerequisite for using the FanAnalog function block

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Date / time of the last operation of the fan dtLastFb¶

The output dtLastFb provides the current date/time during the counting process.
At the output dtLastFb, the date/time at the end of the last counting process is available outside the counting process.
The output dtLastFb outputs the initialization value DT#1970-1-1-1-0.0.0 before the first counting process.

Operation monitoring¶

Operation monitoring can be enabled ( = TRUE ) or disabled ( = FALSE ) by the setpoint / parameter xFbControl.

If monitoring is disabled, the alarm message of the operational monitoring xAlarmFb is always disabled ( = FALSE ).
When monitoring is enabled, the alarm message of the operational monitoring xAlarmFb is activated ( = TRUE ) if the following states are simultaneously present during a minimum duration udiFbControlTime:

xHB and xFb = unequal ( release state and operating state do not match )
eAOO = HVACTYPES.eManBin.Auto ( Manual override - Hardware for enabling in automatic )
xFbControl = TRUE ( Operation monitoring is enabled )

The alarm message of the operation monitoring xAlarmFb is reset ( = FALSE ) if at least one of the following conditions is fulfilled:

eAOO = HVACTYPES.eManBin.Auto ( switch back to automatic mode of manual override - hardware )
xQuit = TRUE ( reset of monitoring is active )
xFbControl = FALSE ( operation monitoring disabled )

low air pressure monitoring¶

Air shortage monitoring is performed by evaluating the states at the inputs xPFb or xFb.

The alarm message of the low air pressure monitoring xAlarmPFb is activated ( = TRUE ), if with enabled monitoring ( xPFbControl = TRUE ) during the message delay of the low air pressure monitoring udiPFbControlTime the input xPFb is permanently not active ( = FALSE ) and the input xPFb is not active ( = FALSE ).
input xFb is active ( = TRUE ).

The alarm message of the low air pressure monitoring xAlarmPFb is deactivated ( = FALSE ) if the conditions for activation are no longer given and at the same time the monitoring is blocked ( xPFbControl = FALSE ) and / or the reset of the error messages ( monitoring ) is active ( xQuit = TRUE ).

Collective alarm signal xAlarm¶

The collective alarm signal is activated ( = TRUE ) if at least one of the conditions described below is fulfilled.

xAlarmIn = TRUE ( Fan fault message is active )
xAlarmFB = TRUE ( error from Operation monitoring )
xAlarmPFB = TRUE ( error from low air pressure monitoring )
xMSwitch = FALSE ( repair switch in position Off )

In all other cases, the collective alarm is not active (= FALSE).

Control signal for fan before manual override rAA¶

The control signal of the fan before manual override rAA is taken from the input control signal - speed rY, if either the output xHB and / or the input xFb is active.
In all other situations, the control signal rAA is set to the value 0.0 %.

Control signal for fan after manual override rHA¶

The control signal of the damper actuator after manual override rHA corresponds to the control signal of the fan before manual override rAA plus a manual override module.

Collective message automatic mode xAuto¶

The collective message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

- Eingang eAOO = HVACTYPES.eManBin.Auto
- Eingang eAO = HVACTYPES.eManNum.Auto
- Sollwert / Parameter eManModeB = eMANBIN.Auto
- Sollwert / Parameter eManModeN = eMANNUM.Auto
