FanControl (FB)¶

FUNCTION_BLOCK FanControl

Short Description¶

Control of a variable speed fan ( controlled variable: absolute pressure, differential pressure )
Additional functions: Manual overriding controls, repair switches, operating hours counters, operation monitoring, air shortage monitoring, limit value monitoring
Typical application: control of a fan with frequency converter

Portrayal¶

Functional Description¶

General¶

This function block controls a variable-speed fan via a continuous control signal rHA.
The general release of the fan takes place via the request xEn and the release signal xHB.
The pressure is controlled by a freely parameterizable PI controller to which the pressure setpoint rSetPoint and the actual pressure value rActValue are fed and from which the control signal of the controller is generated.

Essentially, the following function blocks are used:

ControlPI

ManBin

ManNum

TwoPointFun

Ramp

CTime

AirShortage

OperatingHours

Enable - Fan before manual override xAB¶

The release - fan before manual override xAB is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

Case 1:

xEn = TRUE
xMSwitch = TRUE
xAlarmControl = FALSE

Case 2:

xEn = TRUE
xMSwitch = TRUE
xAlarmControl = TRUE
xAlarm = FALSE

In all other situations the Release of the fan, before manual override xAB, is not activated ( = FALSE ).

Enable - Fan after manual override xHB¶

The release - fan after manual override xHB corresponds to the release - fan before manual override xAB, additionally extended by a manual override module.
The release can generally only take place if the forced locking of the fan is not active after manual override ( xForce = FALSE ).
and the collective alarm message 2 is not active ( = FALSE ).

Repair switch message xMSwitch¶

xMSwitch = TRUE: Repair switch was not triggered ( normal state )
xMSwitch = FALSE: Repair switch was triggered ( error condition)

Forced shut-off of the fan after manual override xForce¶

xForce = FALSE: no foced locking ( normal state )
xForce = TRUE: forced locking ( state of emergency )

Operating hours and switch-on procedures udiOperatingHours and udiCountFb¶

The number of operating hours and the number of switch-on processes are recorded ( = counting process active ) if the fan operating message xFb is active and the collective alarm message xAlarm is not active ( = FALSE ) at the same time.
The results are available at the counter outputs udiOperatingHours and udiCountFb.

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Prerequisite for using the FanControl function block

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Date / time of the last operation of the fan dtLastFb¶

The output dtLastFb provides the current date/time during the counting process.
At the output dtLastFb, the date/time at the end of the last counting process is available outside the counting process.
The output dtLastFb outputs the initialization value DT#1970-1-1-1-0.0.0 before the first counting process.

Operation monitoring xAlarmFb¶

Operation monitoring can be enabled ( = TRUE ) or disabled ( = FALSE ) by the setpoint / parameter xFbControl.

If monitoring is disabled, the alarm message of the operational monitoring xAlarmFb is always disabled ( = FALSE ).
When monitoring is enabled, the alarm message of the operational monitoring xAlarmFb is activated ( = TRUE ) if the following states are simultaneously present during a minimum duration udiFbControlTime:

xHB and xFb = unequal ( release state and operating state do not match )
eAOO = HVACTYPES.eManBin.Auto ( Manual override - Hardware for enabling in automatic )
xFbControl = TRUE ( Operation Monitoring is enabled )

The alarm message of the operation monitoring xAlarmFb is reset ( = FALSE ) if at least one of the following conditions is fulfilled:

eAOO = HVACTYPES.eManBin.Auto ( switch back to automatic mode of manual override - hardware )
xQuit = TRUE ( reset of monitoring is active )
xFbControl = FALSE ( operation monitoring disabled )

Low air pressure monitoring xAlarmPFb¶

Air shortage monitoring is performed by evaluating the states at the inputs xPFb or xFb.

The alarm message of the low air pressure monitoring xAlarmPFb is activated ( = TRUE ), if with enabled monitoring ( xPFbControl = TRUE ) during the message delay of the low air pressure monitoring udiPFbControlTime the input xPFb is permanently not active ( = FALSE ) and the input xPFb is not active ( = FALSE ).
Input xFb is active ( = TRUE ).

The alarm message of the low air pressure monitoring xAlarmPFb is deactivated ( = FALSE ) if the conditions for activation are no longer given and at the same time the monitoring is blocked ( xPFbControl = FALSE ) and / or the reset of the error messages ( monitoring ) is active ( xQuit = TRUE ).

Collective alarm message 1 - Output xAlarm¶

The collective alarm message is activated ( = TRUE ) if at least one of the conditions described below is fulfilled.

xAlarmIn = TRUE ( Fan alarm message is active )
xAlarmFB = TRUE ( error from operation monitoring )
xAlarmPFB = TRUE ( error from low air pressure monitoring )
xMSwitch = FALSE ( repair-switch in Position Off )
xAlarmMaxValue = TRUE ( maximum value exceeded )
xAlarmMinValue = TRUE ( Minimum value fallen short of )

In all other cases, the collective alarm is not active (= FALSE).

Collective alarm message 2¶

The collective alarm message is activated ( = TRUE ) if at least one of the conditions described below is fulfilled.

xAlarmIn = TRUE ( Fan alarm message is active )
xAlarmFB = TRUE ( error from operation monitoring )
xAlarmPFB = TRUE ( error from low air pressure monitoring )
xMSwitch = FALSE ( repair-switch in Position Off )
xAlarmMaxValue = TRUE ( maximum value exceeded )

In all other cases, the collective alarm is not active (= FALSE).

Reset of the alarm messages xQuit¶

Each rising edge ( FALSE > TRUE ) at input xQuit deletes any existing alarm messages from the following monitoring devices

Low air pressure monitoring
Operation monitoring
Minimum value monitoring
Maximum value monitoring

Pressure control process¶


General

The actual pressure value rActValue is scaled by a linear characteristic curve and fed to the PI controller.
The pressure setpoint rSetPont is filtered by a signal ramp, scaled by a linear characteristic and fed to the PI controller.
The output signal of the PI controller is filtered by a further signal ramp and fed to the output for the control signal rHA.


Enable

The control process is enabled with a time delay of udiWaitControl after the active operating message of the fan ( xFb = TRUE ).
The control process is disabled without time delay after the fan has been inactive ( xFb = FALSE ).


Pressure setpoint

The pressure setpoint is set to value 0.0 if the release of the fan is not active ( xHB = FALSE ).
The pressure setpoint value is taken from the input rSetPoint, filtered by a signal ramp and scaled by a linear function defined by 2 points ( prerequisite: xHB = TRUE ).
The relevant parameters of the signal ramp can be set individually ( Maximum variation = (rMaxValue / 5) per 10s ).
The relevant parameters of the linear function can be set individually ( input range in Pa: 0.0 - rMaxValue, output range in %: 0.0 - 100.0 ).


Actual print value

The actual pressure value is taken from input rActValue and scaled by a linear function defined by 2 points.
The relevant parameters of the linear function can be set individually ( input range in Pa: 0.0 - rMaxValue, output range in %: 0.0 - 100.0 ).


Limitation of the control signal of the PI-controller rYMin and rYMax

The control signal of the pressure controller( PI-controller ) is limited by the adjustable limit values rYMin and rYMax ( minimum value monitoring not active ) or rSubstitueHA ( minimum value monitoring active ).


Signal ramp of the control signal from the PI controller

The control signal from the PI controller is filtered by a signal ramp.
The relevant parameters of the signal ramp are individually adjustable ( Maximum variation = rYRampVariation per rYRampTime ).


Parameters of the PI controller

The essential parameters for the control behaviour can be specified externally. They must be adjusted according to the controlled system.
A meaningful basic parameterization has already been carried out, which is sufficient in many applications.

rYMin = Lower absolute limit value of the controller output signal
rYMax = Oberer absoluter Grenzwert des Regler-Ausgangssignals
rOffset = Setpoint offset - Pressure controller
rKp = amplification factor - pressure controller
rTn = reset time - pressure controller in s
eAction = Direction of action of the pressure controller ( Inverted, Direct )
rInitValue = Initialization value for the control signal
rYDisabled = Control signal with inactive pressure controller


Signals of the PI controller

The signals calculated during the control process (before the signal ramp) are available for checking the control process.

rPiDiff = Control deviation at pressure controller
rPiPartP = Proportional component of the control signal
rPiPartI = integral part of the control signal
xPiLimitActiv = Status - Limit value exceeded - Control signal ( FALSE = Off, TRUE = On )

Control signal of the controller

If the pressure controller is not enabled, the deactivation value rYDisabled is transmitted to the output control signal of the controller (prerequisite: ePiManModeN = eMANNUM.Auto ) and limited by the absolute limit values.
The integral component is set to 0.0.

When the pressure controller is enabled, the control signal of the controller is calculated ( precondition: ePiManModeN = eMANNUM.Auto ) and limited by the absolute limit values.

With the basic setting of the direction of action eAction ( = eControlMode.Inverted ), a too low actual pressure value ( actual value < setpoint ) causes a continuous increase in the value of the control signal of the controller.

Control signal for fan before manual override rAA¶

The control signal of the fan before manual override rAA is primarily taken from the output signal of the pressure controller after the signal ramp.
However, this output signal is additionally limited downwards by the lower absolute limit value rYMin during the request of the fan ( *xHB = TRUE ).
If the minimum value is fallen short of ( xAlarmMinValue = TRUE ), the control signal before manual override rAA is replaced by the fixed value rSubstitueHA.

Control signal for fan after manual override rHA¶

The control signal of the fan after manual override rHA corresponds to the control signal of the fan before manual override rAA additionally extended by a manual override module.

Operating mode for the output signal of the controller ePIManModeN¶

The ePiManModeN parameter allows the output signal of the pressure controller to be switched from automatic to manual mode and vice versa.
In manual mode, the output signal of the pressure controller takes over the output signal in manual mode ePiManValue.

Monitoring of the maximum pressure xAlarmMaxValue¶

The output xAlarmMaxValue is activated ( = TRUE ) if the current actual pressure value rActValue exceeds the adjustable maximum value rMaxValue for a minimum period of 10s.
The output xAlarmMaxValue is deactivated ( = FALSE ) if the current actual pressure value rActValue falls below the adjustable maximum value rMaxValue and simultaneously a rising edge ( FALSE > TRUE ) is detected at the input xQuit.

Monitoring of the minimum pressure xAlarmMinValue¶

The output xAlarmMinValue is activated ( = TRUE ), if during a minimum time period udiWaitMinValue ( 1.0 - 600.0 s ) the following conditions are permanently fulfilled at the same time:

xFb = TRUE ( fan is in operation )
eAO = HVACTYPES.eManNum.Auto ( Operating mode of manual override - Hardware for the control signal in automatic mode )
rHA > rMinHAValue ( Control signal is greater than the minimum value )
rActValue < rMinValue ( actual pressure value is smaller than the minimum value )
No rising edge ( FALSE > TRUE ) was detected at the input *xQuit
The output xAlarmMinValue is deactivated ( = FALSE ) if the conditions for activation are no longer fulfilled and at the same time a rising edge ( FALSE > TRUE ) is detected at the input xQuit.

Collective message automatic mode xAuto¶

The collective message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

- Eingang eAOO = HVACTYPES.eManBin.Auto
- Eingang eAO = HVACTYPES.eManNum.Auto
- Sollwert / Parameter eManModeB = eMANBIN.Auto
- Sollwert / Parameter eManModeN = eMANNUM.Auto
- Sollwert / Parameter ePiManModeN = eMANNUM.Auto
