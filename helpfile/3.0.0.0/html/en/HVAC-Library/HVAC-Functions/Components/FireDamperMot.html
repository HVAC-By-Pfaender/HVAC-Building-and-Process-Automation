FireDamperMot (FB)¶

FUNCTION_BLOCK FireDamperMot

Short Description¶

Control of a motorised fire damper with two position feedback signals
Additional functions: Function test of the fire damper
Typical application: Control of a fire damper in an HVAC system

Portrayal¶

Functional Description¶

General¶

This function block controls a binary motorised fire damper ( xEn = TRUE ), monitors the running times during the opening and closing process and can carry out a complete fire damper test.
The function block can be operated either in normal mode or in test mode.
Test mode has a higher priority than normal mode and can be triggered by a rising edge at input xTestEx and / or at setpoint / parameter xRunTest.

Enable opening / closing of fire damper before manual override xAB¶

The opening / closing release of the fire damper before manual override xAB is taken over unchanged from the opening / closing request of the fire damper xEn if the fire damper test is not active.

Release of opening / closing of fire damper after manual override xHB¶

The release opening / closing of the fire damper after manual override xHB corresponds to the release opening / closing of the fire damper before manual override xAB additionally extended by a manual override module

xAlarm collective alarm¶

The collective alarm xAlarm is activated ( = TRUE ) if at least one of the conditions described below is met:

Case 1: The alarm message for the opening feedback xAlarmOpen is active ( = TRUE ).

Case 2: The alarm message for the closing feedback xAlarmClosed is active ( = TRUE ).

Case 3: Normal operation is active and at the same time the fire damper is not fully open ( xOpen = FALSE ).
( Background: Every fire damper that is not fully opened represents a faulty condition during normal operation ).

In all other cases, the xAlarm collective fault is not activated (= FALSE).

Enabling / disabling the alarm message for the opening feedback xAlarmOpen¶

The monitoring of the running time of the damper during the opening process for an adjustable maximum value ( tOpen ) starts if at least one of the following conditions is fulfilled during normal operation:

Case 1: A rising edge ( FALSE > TRUE ) is detected at output xHB ( = rule ).

Case 2: The request for opening is active ( xEn = TRUE ) and at the same time a rising edge ( FALSE > TRUE ) is detected at input xQuit.

Case 3: The output xHB is active ( = TRUE ) and at the same time a falling edge ( TRUE > FALSE ) is detected at the input xOpen.

The alarm message for the opening feedback xAlarmOpen is activated ( = TRUE ) if at least one of the following conditions is fulfilled after the maximum running time tOpen has elapsed with active output ( xHB = TRUE ):

Case 1: The feedback of the complete opening is not active ( xOpen = FALSE ) ( = normal case ).

Case 2: The responses xOpen or xClosed have the same state.

The alarm message for the opening feedback xAlarmOpen is deactivated ( = FALSE ) if a rising edge ( FALSE > TRUE ) is detected at the xQuit input.

Enabling / disabling the alarm message for the feedback of the closing xAlarmClosed¶

The monitoring of the running time of the damper during the closing process for an adjustable maximum value ( tclose ) starts if at least one of the following conditions is fulfilled during normal operation:

Case 1: A falling edge ( TRUE > FALSE ) is detected at output xHB ( = rule case ).

Case 2: The request for closing is active ( xEn = FALSE ) and at the same time a rising edge ( FALSE > TRUE ) is detected at input xQuit.

The alarm message for the closing feedback xAlarmClosed is activated ( = TRUE ) if at least one of the following conditions is fulfilled after the maximum running time tClose has elapsed when the output is not active ( xHB = FALSE ):

Case 1: The feedback of the complete closing is not active ( xClosed = FALSE ) ( = normal case ).

Case 2: The responses xOpen or xClosed have the same state.

The alarm message for the closing feedback xAlarmClosed is deactivated ( = FALSE ) if a rising edge ( FALSE > TRUE ) is detected at input xQuit.

Duration of the last complete opening tLastOpen¶

The measurement of the duration begins each time at least one of the following conditions is met:

Case 1: A rising edge ( FALSE > TRUE ) is detected at input xOpen.

Case 2: A falling edge ( TRUE > FALSE ) is detected at input xOpen.

Case 3: A rising edge ( FALSE > TRUE ) is detected at input xClosed.

Case 4: A falling edge ( TRUE > FALSE ) is detected at input xClosed. ( general case )

Case 5: A rising edge ( FALSE > TRUE ) is detected at output xHB.

Case 6: A falling edge ( TRUE > FALSE ) is detected at output xHB.

At this output, the runtime is available after each rising edge at the feedback of the complete opening xOpen ( FALSE > TRUE ).

Duration of the last complete closing tLastClosed¶

The measurement of the duration begins each time at least one of the following conditions is met:

Case 1: A rising edge ( FALSE > TRUE ) is detected at input xOpen.

Case 2: A falling edge ( TRUE > FALSE ) is detected at input xOpen. ( general case )

Case 3: A rising edge ( FALSE > TRUE ) is detected at input xClosed.

Case 4: A falling edge ( TRUE > FALSE ) is detected at input xClosed. ( general case )

Case 5: A rising edge ( FALSE > TRUE ) is detected at output xHB.

Case 6: A falling edge ( TRUE > FALSE ) is detected at output xHB.

At this output, the runtime is available after each rising edge at the feedback of the complete closing xClosed ( FALSE > TRUE ).

Duration of the current opening / closing in tRunTime¶

At this output, the current runtime is available during each runtime measurement.

Testing the functions of the fire damper ( test operation )¶

The test mode is always generally triggered if at least one of the conditions described below is met:

Case 1: A rising edge ( FALSE > TRUE ) is detected at input xTestEx.

Case 2: The active state ( = TRUE ) is detected at the setpoint / parameter xRunTest. The active state is reset immediately after detection ( = FALSE ).

However, the actual test mode is only performed if the feedback of the automatic mode byAU ( 0 = automatic, >0 = manual mode ) contains the value 0 and at the same time the operating mode of the manual override for the digital output xHB signals the automatic mode ( eMANBIN = eMANBIN.Auto ).

In test mode, the fire damper is closed and then opened, each time with running time monitoring.
At the beginning of the test mode the outputs xAlarmOpen, xAlarmClosed and xTestOK are resetted ( = FALSE ).

Now the control command for closing is triggered and the running time is monitored for exceeding the maximum value tClosed.
If the correct feedback situation ( xClosed = TRUE and xOpen = FALSE ) is reached without exceeding the maximum value, the following is displayed
the output xAlarmClosed is not activated ( = FALSE ).
In all other situations the output xAlarmClosed is activated ( = TRUE ).

Now the control command for the opening is triggered and the running time is monitored for exceeding the maximum value tOpen.
If the correct feedback situation ( xOpen = TRUE and xClosed = FALSE ) is reached without exceeding the maximum value, the following occurs
the output xAlarmOpen is not activated ( = FALSE ).
In all other situations the output xAlarmOpen is activated ( = TRUE ).

The result display for the xTestOK fire damper test is activated ( = test successful ) if neither of the two outputs xAlarmOpen or xAlarmClosed is active ( = TRUE ).

The test mode ends automatically as soon as the fire damper test is completed.

Prerequisites¶

Prerequisite for using the function block FireDamperMot

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.
