OptStartHeat (FB)¶

FUNCTION_BLOCK OptStartHeat

Short Description¶

Switch-on optimisation for heating operation ( principle: variable preheating time )
Target: Room temperature = target temperature at the start of daytime operation
Typical application: Zone control loop with heating valve

Functional Description¶


The output xDaytime is activated with generally enabled switch-on optimization ( xEn = TRUE ) depending on the historical data, the current outdoor temperature, the current room temperature and the current setpoint temperature for the day/comfort mode before the planned start of the comfort/day mode.

This preheating ensures that the room temperature already corresponds to the desired target temperature at the start of daytime operation.
The preheating time required for this is recalculated individually for each preheating process, the actual preheating time is also included in the historical data.
Preheating processes over several days are possible.
The switch-on optimization serves to save energy and increase operating comfort.
The components relevant for switch-on optimization are processed every 60s.

The preheating process is divided into four time periods.
They are run through one after the other during every successful preheating process.
The preheating process can end before, exactly at or after the switch-on point ( = transition to the comfort / day mode ).



Test time (bState = 1)¶

The test time is used for cyclic testing (every 60s) to determine whether the conditions for enabling the preheating have been fulfilled.

It starts if the following conditions are met at the same time:

Condition 1: Preheating process is not active ( xProcess = FALSE )

Condition 2: The week clock is not active ( xActive = FALSE )

Condition 3: A valid next switch-on point exists

It ends according to the rules - if the conditions for enabling preheating are fulfilled ( = start of dead time ).
It ends in any case if the next valid switch-on point is equal to the current time.

Dead time (bState = 2)¶

The dead time is used for preconditioning the heating.

It begins with the regular end of the test period.

It ends according to the rules - if the current room temperature rTi exceeds the room temperature at the beginning of the dead time by the adjustable difference GV_OPTSTARTHEAT.rDTDiffTemp for a period of at least 120s (= beginning of the heating time).

It ends in any case if the next valid switch-on point is equal to the current time.

Heating time (bState = 3)¶

The heating time is used to preheat the zone.

It begins with the regular end of the dead time.

It ends according to rules - if the current room temperature rTI approaches the current room setpoint temperature in comfort mode/day mode rSetTemp permanently for a period of at least 120s except for a difference smaller than the global variable GV_OPTSTARTHEAT.rDiffSetTemp.

It ends if a falling signal edge is detected at the state of the clock channel of the weekly clock xActive.

It ends if the current heating time exceeds the adjustable maximum value uiHTMax.

Heating time end (bState = 4)¶

The end of the heating time serves to complete the successful preheating process. The historical data is updated if the release of the data storage in the historical memory ( xEnAdapt ) is active, provided that no limit values are violated.

It begins with the regular end of the heating period.

Enable xEn¶

This input is used for the general enable of the switch-on optimization ( xEn = TRUE ). In the locked state ( xEn = FALSE ) the outputs are defined as follows:

Release xEnAdapt¶

This input is used to release the data storage in the historical memory ( xEnAdapt = TRUE, prerequisite: xEn = TRUE ).
After each successful preheating process, the relevant parameters are stored and are available again for the next preheating process.
In the locked state ( xEnAdapt = FALSE ) no data storage takes place.

Release xResAdapt¶

This input is used to initialize the historical memory with each rising signal edge ( prerequisite: xEn = TRUE ).

Input rTo¶

This input is used to record the current outdoor temperature.
It is additionally filtered and limited in time for switch-on optimization ( minimum value = GV_OPTSTARTHEAT.rTOMin, maximum value = GV_OPTSTARTHEAT.rTOMax ).

Input rTi¶

This input is used to record the current room temperature.
It is additionally filtered and limited in time for switch-on optimization ( minimum value = GV_OPTSTARTHEAT.rTIMin, maximum value = GV_OPTSTARTHEAT.rTIMax ).

Input rSetTemp¶

This input is used to record the current room setpoint temperature for the comfort / day mode.
Each time the setpoint temperature is changed, it is reset to the basic state ( see Release xEn - locked state ).

Output xDaytime¶

This output is used to enable the comfort/day mode ( Off = FALSE, On = TRUE ) by the switch-on optimization.
It is activated during the transition from the test time to the dead time.
It is deactivated if the next valid switch-on point is reached.

Output xProcess¶

This output is used to display the active preheating process ( Off = FALSE, On = TRUE ).
It is active if the preheating process is in phases 1 - 4 and / or the output xDaytime is active.

Output bState¶

This output is used to display the active phase of the preheating process.

Output wError¶

This output is used to display warnings / errors during the last preheating process.
Each message is assigned a bit ( bit = 0 = no error, bit = 1 = warning / error active ).
This output is permanently updated.
The date / current time is also recorded for each change.
This output is always initialized with 0 ( = no error ) at the beginning of the test time.
Each time an error occurs, the preheating process is stopped and the ‘Off / Deactivated’ phase is initiated.
Each time a warning occurs, the output wError is updated.

Output bResult¶

This output is used to display the quality of the last successful preheating process.
The output is updated as required at the switch-on point ( = transition to comfort / day mode ).
The absolute amount from the calculation of rSetTemp - GV_OPTSTARTHEAT.rDiffSetTemp - rTi is used for the valuation.
This output is always initialized with 0 ( = no result ) at the beginning of the test time.

Output xActive¶

This output is used to display the status of the clock channel of the weekly clock ( Off = FALSE, On = TRUE ).
The weekly clock is permanently updated independently of the state of xEn ( = general enable of switch-on optimization ).
Each time the settings of the weekly clock are changed, the clock is resetted to the basic state ( see Release xEn - Locked state ).

Output uiDeadTime¶

This output is used to display the current dead time duration ( min ).
This output is always initialized with 0 min at the beginning of the test time.

Output uiHeatTime¶

This output is used to display the current heating time ( min ).
This output is always initialized with 0 min at the beginning of the test time.

Output uiNumber¶

This output is used to display the total number of successful preheating processes.

Output rAverageDiff¶

This output is used to display the average absolute room temperature deviation of the last 10 successful preheating processes.
This output is always updated after updating the bResult output.
This output is initialized with 0 K at each rising signal edge at input xResAdapt.

This output can be used in combination with the xResAdapt input to reinitialize the historical data in the event of major deviations.

Step 1: preheating process¶

After the software download or after each rising signal edge at input xResAdapt, the historical data is re-initialized.
The dead times are set with the parameter uiDTMax, the heating gradients with the parameter uiHGMax and the number of successful preheating processes with the value 0.

Parameters uiHGMin¶

This parameter is used to define the minimum permissible heating gradient ( min / K ).
It is used to check whether limit values have been exceeded. In the event of a limit value violation, the historical data is not updated.

Parameters uiHGMax¶

This parameter is used to define the maximum permissible heating gradient ( min / K ).
It is used to check whether limit values have been exceeded. In the event of a limit value violation, the historical data is not updated.

Parameters uiDTMin¶

This parameter is used to define the minimum permissible dead time ( min ).
It is used to check whether limit values have been exceeded. In the event of a limit value violation, the historical data is not updated.

Parameter uiDTMax¶

This parameter is used to define the maximum permissible dead time ( min ).
It is used to check whether limit values have been exceeded. In the event of a limit value violation, the historical data is not updated.

Parameter uiHTMin¶

This parameter is used to define the minimum permissible heating time ( min ).
It is used to check whether limit values have been exceeded.

Parameter uiHTMax¶

This parameter is used to define the maximum permissible heating time ( min ).
It is used to check whether limit values have been exceeded.
Each time a limit value violation occurs, the preheating process is stopped and the ‘Off / Deactivated’ phase is initiated.

This parameter is used to set the weekly clock. The general functionality is described in detail in the description of the function block ‘Clock7d1Ch’.

Global variable GV_OPTSTARTHEAT.rTOMin¶

This global variable is used to define the minimum permissible outdoor temperature ( °C ).
It is used for checking and correcting violations of limit values.

Global variable GV_OPTSTARTHEAT.rTOMax¶

This global variable is used to define the maximum permissible outdoor temperature ( °C ).
It is used for checking and correcting violations of limit values.

Global variable GV_OPTSTARTHEAT.rTIMin¶

This global variable is used to define the minimum permissible room temperature ( °C ).
It is used for checking and correcting violations of limit values.

Global variable GV_OPTSTARTHEAT.rTIMax¶

This global variable is used to define the maximum permissible room temperature ( °C ).
It is used for checking and correcting violations of limit values.

Global variable GV_OPTSTARTHEAT.rDiffSetTemp¶

This global variable is used to determine the deviation from the room target temperature at the end of the preheating process ( K ).
It determines to what extent the room temperature must approach the setpoint temperature in day/comfort mode in order to terminate the active preheating process.

Global variable GV_OPTSTARTHEAT.rDTDiffTemp¶

This global variable is used to define the increase of the room temperature to detect the end of the dead time ( K ).
It determines how far the room temperature must rise permanently during the dead time in order to initiate the heating time.

Global variable GV_OPTSTARTHEAT.uiPHMaxTime¶

This global variable is used to determine the maximum duration of the preheating process.
It is used for internal calculations.

Global variable GV_OPTSTARTHEAT.rLimitResultWell¶

This global variable is used to define the limit value ‘Good preheating process’ for the Display of the quality of bResult.

Global variable GV_OPTSTARTHEAT.rLimitResultNormal¶

This global variable is used to define the ‘Normal preheating process’ limit value for the display of the quality of bResult.
