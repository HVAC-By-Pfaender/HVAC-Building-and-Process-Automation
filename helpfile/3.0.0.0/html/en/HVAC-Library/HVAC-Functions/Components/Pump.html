With the parameter xAlarmControl the behaviour of the output xAB can be influenced in case of an error.
If xAlarmControl = TRUE, the automatic output xAB is switched off in the event of an error (xAlarm = TRUE), if the manual override module is in automatic mode, then the output xHB is also switched off.
With xAlarmControl = FALSE there is no reaction.

Follow-up period¶

The follow-up module is implemented by a switch-off delay with adjustable duration. The input of the follow-up module is active if the enable ‘xEn’ is active and the collective alarm message xAlarm is not active at the same time.

Anti-lock module¶

The anti-lock module is implemented by the binary anti-lock module with adjustable time duration.
The duration tAbsTime, the time tAbsTime and the weekday eAbsDay are adjustable.
The time stamp (date / time) of the last anti-lock operation dtAbsLastOn is available.

Collective message automatic mode xAuto¶

The collective message Automatic mode xAuto is activated ( = TRUE ) if the input eAOO contains the value HVACTYPES.eManBin.Auto and the setpoint / parameter eManModeB contains the value eMANBIN.Auto.
In all other cases, the collective message Automatic mode xAuto is not activated ( = FALSE ).

Forced override xForce¶

The forced override acts directly on the output xHB. The parameter xForceOption defines the state of the output xHB when TRUE at the input xForce.

Operating hours counter¶

The operating hours meter is active if the circulation pump is in operation (xFb = TRUE) and at the same time the collective malfunction *xAlarm*is not active (*xAlarm*= FALSE).
The number of operating hours udiOperatingHours, and the number of times udiCountFb are output to the corresponding outputs.
The time stamp (date / time) of the circulating pump dtLastFb operation is available.

Write access to the counter readings

Via the path ‘Instancename._OperatingHours.udiOperatingHours’ or ‘Instancename._OperatingHours.udiCountFb’ a write access to both counter readings is possible.

Operation monitoring¶

The operation monitoring monitors the correct function of the circulation pump.
The monitoring can generally be enabled (xFbControl = TRUE) or disabled (xFbControl = FALSE).
The automatic operation of the circulation pump must also be active ( eAOO = HVACTYPES.eManBin.Auto ).

The input xFb and the output xHB are permanently compared with each other.
If the states are always unequal during an adjustable period of time udiFbControlTime (i.e. release of the pump but no operation feedback), the alarm message of the operation monitoring xAlarmFb is activated.

The alarm message of the operation monitoring xAlarmFb is reset if the enabling of the monitoring xFbControl is disabled, or the reset input xQuit is active, or the automatic mode of the circulation pump ( eAOO = HVACTYPES.eManBin.Auto ) is activated.
