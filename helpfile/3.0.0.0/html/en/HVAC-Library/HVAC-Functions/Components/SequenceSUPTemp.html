SequenceSUPTemp (FB)¶

FUNCTION_BLOCK SequenceSUPTemp

Short Description¶

Supply air temperature control for a air-handling unit with energy recovery (continuous recirculation damper system and energy recovery) including releases for active heating and cooling.

Typical application: Supply air temperature control in a air-handling unit

Portrayal¶

Functional Description¶

General¶

After enabling ( xOpTCon = TRUE ), this function block enables the supply air temperature rSAT to be controlled to the setpoint rReferenceSAT by generating the control signals rYDamp ( recirculation damper system 0.0 - 100.0 % fresh air proportion ) and rYER ( energy recovery 0.0 - 100.0 % ).
Once the energy recovery systems have been fully utilised, the active heating ( xEnHeat = TRUE ) or active cooling ( xEnCool = TRUE ) is enabled if required.
The control of the supply air temperature with active heating or with active cooling must be carried out via a separate, external PI controller. The control signals of these external PI controllers are set back to the inputs rHeaterYPISAT or rCoolerYPISAT to ensure energy-efficient control of energy recovery.
By taking into account the current outdoor temperature rOAT or exhaust air temperature rRAT, the optimum energy recovery can be achieved for every operating case.
When using this function block, the spatial arrangement of the components (damper system, energy recovery, heater circuit, cooling circuit, etc.) must be taken into account.

Case 1 - Winter case: Supply air temperature < Supply air setpoint temperature, outside temperature < Exhaust air temperature, Supply air setpoint temperature > Exhaust air temperature
Case 2 - Summer case: Supply air temperature > Supply air setpoint temperature, outside temperature > Exhaust air temperature, Supply air setpoint temperature < Exhaust air temperature

Enabling / disabling of the controller xOpTCon¶

In the locked state ( xOpTCon = FALSE ) no control processes are taking into place - the outputs are set to the following signals / states:

rYGen = 100.0 %
xEnHeat = FALSE
xEnCool = FALSE
rYDamp = 100.0 %
rYER = 100.0 %

Enabling / disabling the free night cooling xFNCEn¶

This input signals ( xFNCEn = TRUE ) the active free night cooling and blocks the output for the active cooling xEnCool independently of the other requirements.

Delayed start of the control xPRORamp or xHHWFrost1¶

The internal sequence controller ( PI controller ) is disabled if either the input xPRORamp and / or the input xHHWFrost1 is active ( = TRUE ).
The internal sequence controller is enabled with a time delay of 5 minutes after both inputs are inactive again ( = FALSE ).

Feedback of the control signals rHeaterYPISAT and rCoolerYPISAT respectively¶

The feedback of the control signals for the active heating rHeaterYPISAT and cooling rCoolerYPISAT is monitored internally and evaluated as follows:

Control signal rHeaterYPISAT > 8.0 %: Deactivation of the internal sequence controller
Control signal rHeaterYPISAT < 3.0 %: Activation of the internal sequence controller after a time delay of 500 ms

Control signal rCoolYPISAT > 8.0 %: Deactivation of the internal sequence controller
Control signal rCoolYPISAT < 3.0 %: Activation of the internal sequence controller after a time delay of 500 ms

General control signal from the sequence control rYGen¶

This control signal in the range 0.0 - 100.0 % ( 0.0% = maximum energy recovery, 100.0 = no energy recovery ) is primarily generated by the internal sequence controller and influenced by the input variables rSAT ( supply air temperature ), rReferenceSAT ( supply air setpoint temperature ), rOAT ( fresh air temperature ) and rRAT ( extract air temperature ).

Enabling / disabling the active heating xEnHeat¶

The release of the active heating is active ( = TRUE ), if the following conditions are fulfilled at the same time:

xOptCon = TRUE
( outdoor temperature > exhaust air temperature AND rYGen > 99% ) OR ( outdoor temperature < exhaust air temperature AND rYGen < 1% )

In all other situations the active heating is blocked ( = FALSE ).

Enabling / disabling of active cooling xEnCool¶

The enable of the active cooling is active ( = TRUE ), if the following conditions are fulfilled at the same time:

xOptCon = TRUE
( outdoor temperature > exhaust air temperature AND rYGen < 1% ) OR ( outdoor temperature < exhaust air temperature AND rYGen > 99% )
xFNCEn = FALSE
rAverage2h > rSetpoint_CoolOAT ( blocking of active cooling at low outside temperatures )

In all other situations the active cooling is disabled ( = FALSE ).

Control signal for the damper system rYDamp¶

The control signal for the damper system ( 0.0 - 100.0 %, 0.0 % = no fresh air component, 100.0 % maximum fresh air component ) is produced from the sequence control rYGen as follows, depending on the general control signal:

The control signal rYGen is linearly converted in the range r1Damp to r2Damp into a signal in the range 0.0 - 100.0 and limited.

Control signal for the energy recovery system rYER¶

The control signal for the energy recovery ( 0.0 - 100.0 %, 0.0 % = no energy recovery, 100.0 % maximum energy recovery ) is produced as follows depending on the general control signal from the sequence control rYGen:

The control signal rYGen is linearly converted and limited in the range r1Er to r2Er into a signal in the range 100.0 - 0.0.
