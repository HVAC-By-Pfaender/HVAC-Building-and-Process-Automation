Valve2P (FB)¶

FUNCTION_BLOCK Valve2P

Short Description¶

Control of a binary valve actuator with two position feedback signals

Typical application: Drive of a boiler plant with butterfly valves

Malfunction Position Valve opening xAlarmOpen¶

The Alarm message for the position at the valve opening xAlarmOpen is activated ( xAlarmOpen = TRUE ) if the following conditions are fulfilled at the same time:

Condition 1: The enabling of the xControl position monitoring is enabled (= TRUE)

Condition 2: Automatic mode is selected as operating mode of the manual override- hardware eAOO (eAOO = HVACTYPES.eManBin.auto)

Condition 3: The states of the release of the damper actuator after manual override xHB and the feedback for the damper opening xOpen are continuously different at least during the signal delay of the position monitoring udiControlTime ( xHB = FALSE and xOpen = TRUE ) OR ( xHB = TRUE and xOpen = FALSE ) ).

The alarm message for the position at the valve opening xAlarmOpen is deactivated if at least one of the above conditions is no longer fulfilled.

Malfunction Position valve opening xAlarmClosed¶

The Alarm message for the position at the valve opening xAlarmClosed is activated ( xAlarmClosed = TRUE ) if the following conditions are fulfilled at the same time:

Condition 1: The enabling of the xControl position monitoring is enabled (= TRUE)

Condition 2: Automatic mode is selected as operating mode of the manual override- hardware eAOO (eAOO = HVACTYPES.eManBin.auto)

Condition 3: The states of the release of the valve actuator after manual override xHB and the feedback for valve closure xClosed are the same throughout at least during the message delay of the position monitoring udiControlTime ( ( xHB = TRUE and xOpen = TRUE ) OR ( xHB = FALSE and xOpen = FALSE ) ).

The alarm message for the position at the valve opening xAlarmClosed is deactivated if at least one of the above conditions is no longer fulfilled.

xAlarm collective alarm¶

The collective alarm xAlarm is activated ( = TRUE ) if at least one of the two faults is active for the damper positions ( xAlarmOpen and xAlarmClosed ).
In all other cases, the xAlarm collective fault is not activated (= FALSE).

Anti-lock module¶

The anti-lock module is implemented by the binary blocking protection module with adjustable time duration (FB: AbsB).
The time duration udiAbsTime, the time point todAbsStartTime and the weekday eAbsDay are adjustable.
The minimum run time for locking the anti-lock operation is 30s.
The dtAbsLastOn time stamp (date / time) of the last anti-lock operation is available.

Collective message automatic mode xAuto¶

The collective message Automatic mode xAuto is activated ( = TRUE ) if the input eAOO contains the value HVACTYPES.eManBin.Auto and the setpoint / parameter eManModeB contains the value eMANBIN.Auto.
In all other cases, the collective message Automatic mode xAuto is not activated ( = FALSE ).

Forced override xForce¶

The forced override acts directly on the output xHB. The parameter xForceOption defines the state of the output xHB when TRUE at the input xForce.
