Valve3P (FB)¶

FUNCTION_BLOCK Valve3P

Short Description¶

Control of a 3-point valve drive and calculation of the valve position
In addition, the 3-point drive is protected by a anti-lock function.
Typical application: Control of a 3-point valve drive in a heating circuit

Functional Description¶

General¶

This function component is used to control the position of a 3-point valve drive through the xHBOpen or xHBClose outputs enable - valve opening after manual override.
The current valve position is permanently calculated and is available at the rPos output.
Essentially, the following function blocks are used:

Abs3P

ManBin

Position3P

anti-lock process¶

This process is realised by the monitoring of signals 1 and 2 for activity and, if required, by triggering anti-lock.
The test period covers in each case168 hours (= 1 week).
Signals 1 and 2 are active if they are continuously active in the test period during the minimum operating time of 30.0 secs (only signal 1 or only signal 2 or both).
The anti-lock process is thus prevented, the test time restarting.

Signal 1: ( xYOpen AND eAOOOpen = HVACTYPES.eManBin.Auto ) OR ( eAOOOpen = HVACTYPES.eManBin.ManOn )

Signal 2: ( xYClose AND eAOOClose = HVACTYPES.eManBin.Auto ) OR ( eAOOClose = HVACTYPES.eManBin.ManOn )

If no activity was detected during the test period, the anti-lock process is triggered on the eAbsDay weekday at the todAbsStartTime time.
At the same time, the test period restarts.

The active anti-lock process is divided into two consecutive periods of time (Section 1 and Section 2)
In section 1, the combination xABOpen enable - output = TRUE, xABClose enable - output = FALSE is output during the udiAbsTime time period.
In section 2, the combination xABOpen enable - output = FALSE, xABClose enable - output = TRUE is output during the udiAbsTime time period.

The initial triggering of the anti-lock process may be delayed depending on the relevant settings ( weekday eAbsDay and todAbsStartTime ).

Example

Testing period completed on Tuesday morning, start of the anti-lock process not until next Monday.

At the dtAbsLastOn output, the start date and the start time of the last anti-lock process are available.

Prerequisite for the use of the Abs3P functional component

For correct function, the use of the function block TimeRead with xEn = TRUE is mandatory.

Forced control of manual overrides of the outputs¶

If the operating mode of the manual override - valve opening eManModeOpen is in the state eMANBIN.On ( = manual override opening active ), then the operating mode of the manual override - valve closure eManModeClose is inevitably set to the state eMANBIN.Off ( = manual override closing not active ).

If the operating mode of the manual override - valve closing eManModeClose is in the state eMANBIN.On ( = manual override closing active ), then the operating mode of the manual override - valve opening eManModeOpen is inevitably set to the state eMANBIN.Off ( = manual override opening not active ).

rPos current valve position¶

It is used to display the current position of the 3-point actuator in the range 0 - 100% and is calculated from the duration of the signals at the outputs enable - valve opening after manual override xHBOpen and enable - valve closing after manual override xHBClose in combination with the running time of the actuator tDuration.

The position display is independent of the signal generation, i.e. if the position display reaches an end position ( 0% or 100% ), signals are still generated at the outputs xABOpen and xABClose or xHBOpen and xHBClose.
The position indicator does not take into account the possible position changes caused by the manual override - hardware.

Collective message automatic mode xAuto¶

The collective message automatic mode xAuto is activated ( = TRUE ) if the following conditions are fulfilled at the same time:

Input eAOOUp = HVACTYPES.eManBin.Auto
Input eAOODown = HVACTYPES.eManBin.Auto
Setpoint / Parameter eManModeUp = eMANBIN.Auto
Setpoint / Parameter eManModeDown = eMANBIN.Auto

Forced override xForce¶

The forced override acts directly on the outputs xHBOpen and xHBClose. The parameter xForceOption defines the state of the output xHB when TRUE at the input xForce.

With xForceOption = FALSE
If input xForce is TRUE, output xHBOpen is set to FALSE and output xHBClose is set to TRUE.

With xForceOption = TRUE
If the input xForce is set to TRUE, the output xHBOpen is set to TRUE and the output xHBClose is set to FALSE at the same time.
