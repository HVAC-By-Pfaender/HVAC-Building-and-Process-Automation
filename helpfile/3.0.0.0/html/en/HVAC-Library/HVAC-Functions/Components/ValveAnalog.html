ValveAnalog (FB)¶

FUNCTION_BLOCK ValveAnalog

Short Description¶

Control of a continuous valve drive with anti-lock function and continuous position feedback

Typical application: Control valve in a heating circuit

Functional Description¶

rAA control signal before manual override¶

The control signal before manual override rAA is influenced by the anti-lock module.
The anti-lock module is parameterized by the setpoints / parameters udiAbsTime, tAbsStartTime and eAbsDay, the position feedback from the control valve rU is used as input signal.
The minimum change in valve position to lock the blockin protection function is fixed at 10.0%.
The rAA control signal before manual override is limited by ​​**rAbsMinLimit** and rAbsmaxLimit, the two absolute limit values.
The rU position feedback from the control valve can be replaced by the rY control signal from regulation if no position feedback is available.

rHA control signal after manual override¶

The control signal after manual override rHA is influenced by the output of the anti-lock module, the control signal from control rY, manual override and override control.

position monitoring¶

The position monitoring can be enabled (**xControl**= TRUE) or disabled (**xControl**= FALSE).
When position monitoring is enabled, the amount of the difference between the control signal after manual override rHA and the position feedback signal from the control valve rU is permanently determined and compared with the limit value rControlDiff.
If this difference permanently exceeds the limit value during a message delay udiControlTime, an alarm signal is generated if the feedback - automatic mode eAO signals automatic mode ( eAO = HVACTYPES.eManNum.Auto ) at the same time.
This alarm signal is displayed in plain text with the highest priority at the sStatus output and simultaneously transmitted to the xAlarm output ( xAlarm = TRUE ).
The alarm signal is deactivated again ( xAlarm = FALSE ) as soon as the difference falls below the limit value again for the first time.

rU position feedback from the control valve¶

The rU position feedback from the control valve can be replaced by the rY control signal from regulation if no position feedback is available.

Anti-lock module¶

The anti-lock module is implemented by the numerical anti-lock module with adjustable time duration (FB: AbsA).
The time duration udiAbsTime, the time point todAbsStartTime and the weekday eAbsDay are adjustable.
The minimum run time for locking the anti-lock operation is 30s.
The time stamp (date / time) of the last anti-lock operation dtAbsLastOn is available.

Collective message automatic mode xAuto¶

The collective message Automatic mode xAuto is activated ( = TRUE ) if the input eAO contains the value HVACTYPES.eManNum.Auto and the setpoint / parameter eManModeN contains the value eMANNUM.Auto. In all other cases, the collective message Automatic mode xAuto is not activated ( = FALSE ).
In all other cases, the collective message Automatic mode xAuto is not activated ( = FALSE ).

Forced override xForce¶

The forced override affects the output rHA directly. The parameter xForceOption defines the state of the output rHA when TRUE at the input xForce.

With xForceOption = FALSE
If the input xForce is TRUE, the output rHA is set to 0.

With xForceOption = TRUE
If the input xForce is TRUE, the output rHA is set to 100.
