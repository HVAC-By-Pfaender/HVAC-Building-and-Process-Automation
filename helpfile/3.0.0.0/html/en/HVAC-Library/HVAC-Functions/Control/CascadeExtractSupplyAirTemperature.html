CascadeExtractSupplyAirTemperature (FB)¶

FUNCTION_BLOCK CascadeExtractSupplyAirTemperature

Short Description¶

Formation of a supply air setpoint temperature by means of a cascade controller ( PI controller )
Additional functions: Limitation of the supply air setpoint temperature
Typical application: Supply air temperature control in a ventilation system

Portrayal¶

Functional Description¶

General¶

This function block is used to form the target supply air temperature of a ventilation system consisting of

Part 1: Current setpoint temperature ( exhaust air or room temperature ) in °C

Part 2: Current actual temperature ( exhaust air or room temperature ) in °C

taking into account the relative ( rMinSUPTemp or rMaxSUPTemp ) and absolute ( rMinSUP or rMaxSUP ) limits.
In addition, the special operating modes ‘rapid heating’ and ‘free night cooling’ are taken into account when forming the supply air setpoint temperature.
The supply air setpoint temperature is formed with a time delay ( tDelayControl ) after the controller has been enabled / disabled ( xEn = TRUE ).

The cascade controller is enabled if the following conditions described below are simultaneously fulfilled:

Condition 1: The special operating mode fast heating is disabled ( xFastHeatEn = FALSE ).

Condition 2: The special operating mode free night cooling is disabled ( xFNCEn = FALSE ).

Condition 3: The current actual temperature rActTemp does not violate its absolute limits ( +5.0 - +50.0 °C ).

Condition 4: The control is enabled ( xEn = TRUE ) and the switch-on delay ( duration = tDelayControl ) is finished.

In all other situations the cascade controller is disabled.

Control¶

The current actual temperature rActTemp is permanently compared with the current setpoint temperature rSetPtTemp when the cascade controller is enabled ( PI controller, xEn = TRUE, elapsed time tDelayControl, no limit value violation at input rActTemp ) and an offset temperature is generated in the cascade controller depending on the amount / sign of the difference or the controller settings ( rPI_Kp or rPI_Tn ).

In heating mode ( rActTemp < rSetPtTemp ) a positive offset temperature in the range 0.0 - rMaxRelativeSUP K results.
In cooling mode ( rActTemp > rSetPtTemp ) a negative offset temperature in the range 0.0 - rMinRelativeSUP K occurs.
In the fully regulated state ( rActTemp = rSetPtTemp ) the offset temperature does not change.

This offset temperature is then added to the current setpoint temperature and, if necessary, adjusted to the calculated limit values.
rMinSUPTemp or rMaxSUPTemp limited.

At the output rMinSUPTemp the calculated minimum supply air setpoint temperature ( rActTemp + rMinRelativeSUP, absolutely limited to rMinSUP or rMaxSUP ) is permanently displayed.
At the output rMaxSUPTemp the calculated maximum supply air setpoint temperature ( rActTemp + rMaxRelativeSUP, absolutely limited to rMinSUP or rMaxSUP ) is permanently displayed.

The current actual temperature rActTemp is monitored for compliance with limit values.
The cascade controller is deactivated for each limit violation ( rActTemp < +5.0 °C or rActTemp > +50.0 °C ).
