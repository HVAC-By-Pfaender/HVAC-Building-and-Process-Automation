Functional Description¶

General¶

The operating mode selector is used to switch between eMaxMode +1 different operating modes (automatic, manual OFF and eMaxMode -1 manual operating modes).
The switching process is triggered by a rising edge at the setpoint / parameter xOpModeToggle.
The changeover is endless, i.e. after the last manual operating mode, the automatic operating mode is selected with the next changeover process.
The operating mode can also be explicitly selected by selecting the reference / parameter eMode in the range eOpMode.Mode0 - eMaxMode.
The current operating mode is represented numerically by the setpoint/parameter eMode in the range eOpMode.Mode0 - eMaxMode.
It is additionally represented by the corresponding digital outputs xAutoM0, xManOffM1, xManM2, xManM3, etc.

Variable xIn¶

In xActive

Note
FALSE FALSE

Only valid in automatic mode ( eMode = eOpMode.Mode0 )
TRUE TRUE

Only valid in automatic mode ( eMode = eOpMode.Mode0 )

Output variable xActive¶

In automatic mode ( eMode = eOpMode.Mode0 ) the output variable xActive follows the input variable xIn.
With all other operating modes, the status FALSE is always active.

Reset to automatic mode¶

Each rising edge on the setpoint/parameter xResAuto activates automatic mode if the setpoint/parameter xEn is in the TRUE state.

Enabling/disabling switching¶

The operating mode can only be changed if the setpoint/parameter xEn is in the TRUE state.

Switching delay tToggleTime¶

The time between the activation of the operating mode changeover and the actual changeover can be set via the setpoint / parameter tToggleTime.
The time will only start if the operating modes have been switched using the xOpModeToggle setpoint/parameter.

Enter FB Instance OpModeSelect here Example PRG.OpModeSelect

It is sufficient to place the visualization element OpModeSelect_Button. A click on it opens the dialog OpModeSelect in the visualization. The number of operating modes is defined in the FB HVAC.OpModeSelect (eMaxMode) as well as the titles (sTitelM0 to M5) and the dialog title (sTitle and sTitle2). The visualization element automatically adapts to the size of the maximum operating modes. | Note: VAR_IN_OUT interfaces must be assigned, VAR_INPUT are optional. | |
