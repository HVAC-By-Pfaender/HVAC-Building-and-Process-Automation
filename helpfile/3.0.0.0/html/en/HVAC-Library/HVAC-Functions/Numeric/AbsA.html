Functional Description¶

General¶

This function block is used to monitor the position feedback rPosFb on activity. The test period covers 168 hours (= 1 week).
The position feedback rPosFb is active if its value has changed in the check period by an amount greater than the minimum position change rMinChange.
The anti-lock process is thus prevented (= anti-lock process not active). The test time starts again.

If no activity was recorded during the test period, the anti-lock process is triggered with the duration tTime on the weekday eDay at the time todStartTime. ( = anti-lock process active ).
At the same time, the test period starts again.

During the anti-lock process, different signals are shown at the analog output rOut depending on the current value at the position feedback rPosFb, at the beginning of the anti-lock process.

Condition 1: rPosFb > 51% of the upper limit for the analog - output rOut:

rOut = rMinLimit

Condition 2: rPosFb <= 51% of the upper limit for the analog output rOut:

rOut = rMaxLimit

If the anti-lock process is disabled, the analog output rOut shows 0.0.

The first-time triggering of the anti-lock process can be delayed depending on the relevant settings ( weekday eDay and todStartTime ) ( example: testing period finished on Tuesday morning, triggering of the anti-lock process not until next Monday ).

The Output dtAbsLastOn provides the start date and time of the last anti-lock process.

Prerequisite for using the function block AbsA

For correct function the use of the function block TimeRead with xEn = TRUE is necessary.

Display of the anti-lock process xAbsOn¶

This output is used to display an active anti-lock process (xAbsOn = TRUE).

Setpoint / Parameter eDay¶

The anti-lock function can generally be disabled by setting eDay = eDow.Inactive.

Setpoint / Parameter**tTime**¶

The anti-lock function can generally be disabled by setting tTime = 0s.
